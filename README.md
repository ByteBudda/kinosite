📖 О проекте

KINO.QUANTOMIX – это веб-приложение для просмотра фильмов и сериалов с интеграцией TMDB (база данных о фильмах) и Jackett (поиск торрентов).
Вы можете искать контент, смотреть трейлеры, находить раздачи и открывать их через AceStream (или скачивать торрент/магнит).

Приложение имеет:

· Авторизацию по паролю (простая защита от посторонних).
· Тёмную и светлую тему.
· Категории: популярное, лучшее, жанры.
· Глобальный поиск по торрентам через Jackett.
· Встроенный прокси для постеров TMDB.
· Кэширование запросов к TMDB для ускорения работы.

🚀 Установка (для новичков)

1. Установите Node.js

Скачайте LTS-версию с официального сайта.
После установки проверьте в терминале:

```bash
node -v
npm -v
```

2. Скачайте проект

```bash
git clone https://github.com/ваш-логин/kinosite.git
cd kinosite
```

Если Git не установлен, просто скачайте ZIP-архив с GitHub и распакуйте.

3. Установите зависимости

```bash
npm install express axios cors express-session flat-cache
```

⚙️ Настройка

1. Получите ключи API

· TMDB API Key – themoviedb.org → настройки → API.
· Jackett – установите Jackett. После запуска он будет доступен по адресу http://127.0.0.1:9117.
    В интерфейсе Jackett найдите API Key (в настройках).

2. Измените конфигурацию

Откройте файл server.js и замените следующие строки на свои данные:

```js
const SITE_PASSWORD = 'kino-password';      // придумайте свой пароль
const TMDB_KEY = 'ваш_ключ_tmdb';
const J_URL = 'http://127.0.0.1:9117';
const J_KEY = 'ваш_ключ_jackett';
```

Совет: для безопасности лучше вынести эти параметры в переменные окружения (см. раздел «Продвинутая настройка» ниже).

▶️ Запуск

Обычный запуск (для тестирования)

```bash
node server.js
```

Сервер запустится на порту 3000: http://localhost:3000

Запуск через PM2 (для продакшена, автозапуска и стабильности)

PM2 – менеджер процессов для Node.js. Он перезапустит приложение в случае сбоя и может автоматически запускать его при загрузке системы.

1. Установите PM2 глобально

```bash
npm install -g pm2
```

2. Запустите сервер через PM2

```bash
pm2 start server.js --name "kinosite"
```

Флаг --name задаёт имя процесса для удобства.

3. Проверьте, что приложение работает

```bash
pm2 list
```

Вы увидите список запущенных процессов, их статус и потребление ресурсов.

4. Настройте автозапуск PM2 при загрузке системы

```bash
pm2 startup
```

Эта команда выведет инструкцию (например, для systemd). Выполните предложенную команду (скорее всего, она будет начинаться с sudo).
Затем сохраните текущий список процессов:

```bash
pm2 save
```

Теперь после перезагрузки компьютера PM2 автоматически запустит ваш сервер.

5. Полезные команды PM2

· Посмотреть логи: pm2 logs kinosite
· Перезапустить: pm2 restart kinosite
· Остановить: pm2 stop kinosite
· Удалить из списка: pm2 delete kinosite
· Подробная информация: pm2 info kinosite

🧭 Как пользоваться

· Вкладки «Кино» / «Сериалы» / «Глобал» – переключение режимов.
· Боковое меню – категории (популярное, лучшее, жанры).
· Поиск – ищет фильмы/сериалы по названию (TMDB) или торренты (в режиме «Глобал»).

Просмотр фильма

1. Нажмите на карточку.
2. В модальном окне появятся найденные торренты.
3. Для каждого торрента:
   · Магнит / Торрент – скачать.
   · Трекер – открыть страницу раздачи.
   · Смотреть – открыть ссылку AceStream (если установлен клиент AceStream).

⚠️ Возможные проблемы и их решение

Проблема Решение
Ошибка Cannot find module 'express' Запустите npm install в папке проекта.
Не работает поиск торрентов Проверьте, что Jackett запущен и ключ в J_KEY верный.
Постеры не отображаются Проверьте интернет-соединение, возможно, прокси-изображений блокируется антивирусом.
После перезагрузки сервер не запускается Выполните pm2 startup и pm2 save, как описано выше.

🔧 Продвинутая настройка (переменные окружения)

Чтобы не править код, используйте .env:

1. Установите dotenv: npm install dotenv
2. Создайте .env:
   ```ini
   SITE_PASSWORD=kinositeimba
   TMDB_KEY=ваш_ключ_tmdb
   J_URL=http://127.0.0.1:9117
   J_KEY=ваш_ключ_jackett
   ```
3. В начале server.js добавьте require('dotenv').config();
4. Замените переменные на process.env.SITE_PASSWORD и т.д.
